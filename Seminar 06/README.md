# 07. Предефиниране на оператори. 🔧 Приятелски класове и функции. 🫂

Предефинирането на оператори в C++ ни позволява да зададем поведение на вече съществуващи оператори, когато работим с **потребителски дефинирани типове**.

Например, ако имаме клас `Complex` представляващ едно комплексно число, можем да дефинираме какво означава:

- събиране на две комплексни числа с `+`
- сравнение с `==`, `>`, `>=`, `<`, `<=`, `!=`
- извеждане към поток с `<<`
- прочитане от поток с `>>`

Oператорите са обикновени функции със специално име - ключовата дума `operator`, последвана от съответния оператор, и специален начин на извикване.

Както всяка друга функция, и те имат:
- тип на връщане
- параметри
- тяло

---

## Какво е предефиниране на оператори

Идеята е проста: искаме обекти от наш клас да могат да участват в изрази по естествен начин.

Пример:
```cpp
Complex a(1, 2);
Complex b(3, 4);
Complex c = a + b;
std::cout << c;
```

Ако не сме дефинирали `operator+` и `operator<<`, този код няма да работи.

_Предефинирането на оператори **не създава нови оператори**, а само дава смисъл на вече съществуващи оператори за наш тип. Тоест не можем да дефинираме оператор #@ например._

---

### Видове оператори спрямо брой аргументи

В C++ операторите могат да бъдат:

- **унарни** — работят с един аргумент
- **бинарни** — работят с два аргумента
- **тернарни** — работят с три аргумента

#### Примери:

Унарни:

- `-a`, `+a`
- `!a`
- `++a`, `a++`


Бинарни:

- `a + b`
- `a == b`
- `a += b`
- `a = b`

Тернарен:
```cpp
condition ? x : y
```
---

#### Как се подават аргументите:

При бинарен оператор:

```cpp
a + b
```
- `a` е лява операнда
- `b` е дясна операнда

Ако операторът е реализиран като **външна функция**, тогава двете операнди са параметри:

```cpp
Complex operator+(const Complex& lhs, const Complex& rhs);
```

Ако операторът е реализиран като **член-функция**, тогава лявата операнда е обектът `*this`, а като параметър остава само дясната:

```cpp
class Complex {
public:
    Complex operator+(const Complex& other) const;
};
```

Тук извикването `a + b` е еквивалентно на:

```cpp
a.operator+(b)
```
---

## Ограничения

Предефинирането на оператори има важни ограничения.

### Можем да предефинираме само вече съществуващи оператори

Не можем да измисляме нови символи като например:
```cpp
a *** b
a <-> b
@$a
```
### Поне един от операндите трябва да е потребителски дефиниран тип

Позволено:
```cpp
Complex operator+(int, const Complex&);
```cpp

Непозволено:
```cpp
int operator+(int, double);
```
Защото тук и двата типа са вградени.

### Не можем да променяме:

- *приоритета* на оператора
- *асоциативността* му
- *броя аргументи*
- *позицията му* спрямо аргументите

---

## Кои оператори **не могат** да бъдат предефинирани

Следните оператори не могат да се предефинират:

- `?:`
- `sizeof`
- `::`
- `.`
- `*` за дерефериране

---

## Приоритет, асоциативност и позиция

Операторите се характеризират с:

- **приоритет**
- **асоциативност**
- **позиция спрямо аргументите**

### Приоритет

Определя кой оператор се изпълнява пръв в по-сложен израз.

Пример:

```cpp
a + b * c
```
Първо се изпълнява `*`, после `+`.

### Асоциативност

Когато имаме няколко оператора с един и същи приоритет, асоциативността определя реда.

Пример с измислен бинарен оператор `~`:

- ако е лявоасоциативен:

```cpp
a ~ b ~ c ~ d
```

се чете като:

```cpp
(((a ~ b) ~ c) ~ d)
```

- ако е дясноасоциативен:

```cpp
a ~ b ~ c ~ d
```

се чете като:

```cpp
(a ~ (b ~ (c ~ d)))
```

| Оператор                                       | Асоциативност                   |
|------------------------------------------------|---------------------------------|
| `()` , `[]` , `->` , `.` , `++` , `--` (постфиксни) |         лявоасоциативен       |
|  `+`, `-` (унарни), `!`, `++` ,`--` (префиксни) |         дясноасоциативен      |
|  `+` , `-`, `*` , `/` , `%` (бинарни)        					 |      лявоасоциативен|
|    `<<` , `>>`      							 |      лявоасоциативен|
|  `<` , `<=` , `>` , `>=` , `==` , `!=`         |      лявоасоциативен|
|      `&` , `^` , `\|` , `&&` , ` \|\|`   			             |     лявоасоциативен |
|      `?:`   									 |      дясноасоциативен         |
|     `=` , `+=` , `-=` , `*=` , `/=` , `%=` , `>>=` , `<<=` , `&=` , `^=`    |      дясноасоциативен         |

### Позиция

Операторите могат да бъдат:

- **префиксни** — преди аргумента: `++a`, `+a`, `-a`
- **инфиксни** — между аргументите: `a + b`
- **постфиксни / суфиксни** — след аргумента: `a++`

Някой оператори могат да бъдат с различнa позиция, като например `++` и `+`, в зависимост от начина, по който се използват.

---

### Общи правила за операторите при дизайн на клас

При проектиране на клас често се следват тези насоки:

- ако класът има смислен вход/изход, дефинираме `<<` и `>>`
- ако има смислена проверка за равенство, дефинираме `==`
- ако дефинираме `==`, обикновено дефинираме и `!=`
- ако има логика да се сравнява, дефинираме сравнения
- ако дефинираме `+`, обикновено дефинираме и `+=` (аналогично за `-`, `*`, `/`)

---

### Какъв трябва да е типът на връщане за даден оператор

Добре е поведението да следва това на вградените типове.

Обичайни правила:

- аритметичните оператори като `+`, `-`, `*`, `/` връщат **нова стойност**
- съставните оператори като `+=`, `-=`, `*=`, `/=` връщат **референция към левия обект**
- релационните и логическите оператори връщат `bool`
- операторите за вход/изход връщат поток по референция

---

## Извикване на оператор

Ако имаме:

```cpp
a + b
```

това може да се напише и така:

```cpp
operator+(a, b);
```

Ако операторът е член-функция:

```cpp
a += b;
```

е еквивалентно на:

```cpp
a.operator+=(b);
```

Тази форма рядко се използва в практиката, но е полезно да знаем какво реално се случва, и че реално погледнато тези оператори са просто функции.

---

### Кога операторът е член-функция и кога външна функция

#### Задължително като член-функции:

Следните оператори **трябва** да бъдат член-функции:

- `operator=`
- `operator[]`
- `operator()`
- `operator->`

#### Обикновено като член-функции:

Оператори, които променят състоянието на обекта, често са членове:

- `+=`
- `-=`
- `*=`
- `/=`
- префиксен и постфиксен `++`
- префиксен и постфиксен `--`

#### Често като външни функции:

Симетрични оператори, при които и двата операнда са равнопоставени, обикновено е по-добре да са външни:

- `+`
- `-`
- `*`
- `/`
- `==`
- `!=`
- `<`
- `<=`
- `>`
- `>=`

---

#### Задължително външни:

- `<<`
- `>>`

Причината операторите за вход/изход да не бъдат член-функции, е че лявата операнда винаги трябва да е обект от тип std::ostream& / std::istream&. Когато дефинираме един оператор като член-функция, левия операнд става обектът, в който е дефиниран оператора, което не е коректно за `<<` и `>>`.

Затова операторите за форматиран вход / изход, ще дефинираме така:

```cpp
class MyClass { /* ... */ };

std::ostream& operator<<(std::ostream& os, const MyClass& obj);
std::istream& operator>>(std::istream& is, MyClass& obj);
```

### Защо това е добър подход

- няма дублиране на логика
- `+=` съдържа реалното поведение
- `+` просто използва `+=`
- кодът е по-лесен за поддръжка

Същата идея важи и за:

- `-` чрез `-=`
- `*` чрез `*=`
- `/` чрез `/=`

---

### Префиксен и постфиксен инкремент

Това е класически пример, при който трябва да различаваме две версии на оператора.

#### Префиксен `++x`

Връща променения обект.
```cpp
class Counter {
private:
    int value = 0;

public:
    Counter& operator++() {
        ++value;
        return *this;
    }
};
```

#### Постфиксен `x++`

Връща старата стойност отпреди промяната.
```cpp
class Counter {
private:
    int value = 0;

public:
    Counter operator++(int) {
        Counter old(*this);
        ++value;
        return old;
    }
};
```

#### Защо има параметър `int` (dummy parameter)

Той е само за разграничаване на сигнатурата.
Не носи реална стойност и почти никога не се използва. Идеята му е да помогне на компилатора да разграничи дефиницията на префиксния и постфиксния оператор за инкрементация / декрементация.

## Приятелски функции и приятелски класове


### Основна идея

Приятелските функции/класове са външни функции/класове, които имат достъп до private и protected членовете на даден клас. Това е полезно, когато дадена операция логически не принадлежи само на самия обект, но все пак има нужда от достъп до вътрешното му състояние. Това става използвайки ключовата дума `friend`.

---

## Приятелски функции

Типичен пример за приятелска функция е операторът за вход `operator>>`. Той не може да бъде член-функция на нашия клас, защото левият му операнд е поток от тип `std::istream&`. Затова го дефинираме като външна функция, а ако трябва директно да достъпва private полета, я декларираме като приятелска.

Пример:

```cpp
#include <iostream>

class Point {
private:
    int x;
    int y;

public:
    Point(int x = 0, int y = 0) : x(x), y(y) {}

    friend std::istream& operator>>(std::istream& is, Point& p);
};

std::istream& operator>>(std::istream& is, Point& p) {
    // instead of reading a int variable, and then calling a setter
    is >> p.x >> p.y;
    return is;
}
```

Така не се налага да правим излишни публични setter-и само за да можем да четем от поток.


## Приятелски класове

Можем да направим и цял клас приятел на друг клас.

```cpp
class A {
    friend class B;
private:
    int secret = 42;
};
```

Тогава класът `B` има достъп до private и protected членовете на `A` в своите функции.

### Важни свойства

Приятелството:

- **не се наследява**
- **не е транзитивно**
- **не е симетрично**

Тоест:

- ако `B` е приятел на `A`, това не значи, че `A` е приятел на `B`
- ако `C` е приятел на `B`, това не значи, че `C` е приятел на `A`

### Къде се декларира приятелство

Може да се напише в:

- `private`
- `protected`
- `public`

На практика най-често се поставя в `public` частта, за да е по-видимо.

---

## Оператори за форматиран вход и изход

C++ използва `<<` и `>>` за форматиран вход и изход при работа с потоци.

Примери:
```cpp
std::cout << x;
std::cin >> x;
```
Ако искаме:

```cpp
Complex c;
std::cin >> c;
std::cout << c;
```
трябва да предефинираме тези оператори.

### Предефиниране на `operator<<` и `operator>>`

```cpp
class Complex {
private:
    double real;
    double im;

public:
    Complex(double real = 0, double im = 0)
        : real(real), im(im) {}

    friend std::ostream& operator<<(std::ostream& os, const Complex& c);
    friend std::istream& operator>>(std::istream& is, Complex& c);
};

std::ostream& operator<<(std::ostream& os, const Complex& c) {
    return os << c.real << " + " << c.im << "i";
}

std::istream& operator>>(std::istream& is, Complex& c) {
    return is >> c.real >> c.im;
}
```

### Защо тези оператори са външни

Защото при израза:
```cpp
std::cout << c;
```
левият операнд е `std::ostream`, а не `Complex`.

Ако `operator<<` беше член-функция на `Complex`, тогава бихме писали нещо като:
```cpp
c << std::cout;
```
което не е естествено и не съвпада със стандартния синтаксис.

---

### Какво става, ако `operator<<` е вътрешен

```cpp
class A {
private:
    int n;

public:
    A(int n) : n(n) {}

    std::ostream& operator<<(std::ostream& os) {
        os << n;
        return os;
    }
};
```

Тогава ще можем да пишем:
```cpp
A a(42);
a << std::cout;
```

или:
```cpp
a.operator<<(std::cout);
```
но няма да можем да пишем:
```cpp
std::cout << a;
```
а именно това е желаното поведение.

## Оператор за индексация (`operator[]`)

Операторът [] позволява обект от даден клас да се използва по начин, подобен на масив или колекция. Чрез него можем да осигурим достъп до елемент по подаден индекс.

Този оператор се използва най-често в класове, които представляват колекции от елементи — например динамични масиви, вектори, матрици, низове и други структури от данни. Идеята е потребителят на класа да може да пише:

```cpp
Collection c;
// ...
for (int i = 0; i < c.getSize(); i++) {
    cout << c[i];
}
```

Обикновено за `operator[]` се дефинират по две версии, за да може той не само да се чете, но и да се променя:

- версия за неконстантни обекти (модификация)
- версия за константни обекти (четене/обхождане)

Така можем да поддържаме както четене, така и модификация, без да нарушаваме const коректността.

При колекция от примитивни типове:

```cpp
#include <iostream>
#include <stdexcept>

class IntCollection {
private:
    int* data;
    size_t size;

public:
    // ...
    int& operator[](size_t index) {
        if (index >= size) {
            // index validation
            throw std::out_of_range("Index out of range");
        }
        return data[index];
    }

    const int& operator[](size_t index) const {
        if (index >= size) {
            // index validation
            throw std::out_of_range("Index out of range");
        }
        return data[index];
    }
    // for the const one for primitive types such as int,
    // you can also do just int operator[](size_t index) const;

    // ...
};
```

При колекция от обекти:

```cpp
#include <iostream>
#include <stdexcept>

class Student {
private:
    int fn;

public:
    Student(int fn = 0) : fn(fn) {}
};

class StudentCollection {
private:
    Student* data;
    size_t size;

public:

    //...

    Student& operator[](size_t index) {
        if (index >= size) {
            // index validation
            throw std::out_of_range("Index out of range");
        }
        return data[index];
    }

    const Student& operator[](size_t index) const {
        if (index >= size) {
            // index validation
            throw std::out_of_range("Index out of range");
        }
        return data[index];
    }

    //...
};
```

## Предефиниране на оператори за равенство

Операторът `==` трябва да отговаря на очакването за **логическа еквивалентност**.

Пример:
```cpp
class Complex {
private:
    double real;
    double im;

public:
    Complex(double real = 0, double im = 0)
        : real(real), im(im) {}

    friend bool operator==(const Complex& lhs, const Complex& rhs);
    friend bool operator!=(const Complex& lhs, const Complex& rhs);
};

bool operator==(const Complex& lhs, const Complex& rhs) {
    return lhs.real == rhs.real && lhs.im == rhs.im;
}

bool operator!=(const Complex& lhs, const Complex& rhs) {
    return !(lhs == rhs);
}
```

### Добри правила за `==`

Ако дефинираме `operator==`, той трябва наистина да означава:

> “тези два обекта са еквивалентни”

### Полезни свойства

Добре е `==` да бъде:

- **рефлексивен**: `a == a`
- **симетричен**: ако `a == b`, тогава `b == a`
- **транзитивен**: ако `a == b` и `b == c`, тогава `a == c`

## Предефиниране на релационни оператори

Ако даден клас има естествена наредба (искаме да може да подреждаме/сортираме обекти от този клас), можем да дефинираме:

- `<`
- `<=`
- `>`
- `>=`

Пример:
```cpp
class Point {
private:
    int x;
    int y;

public:
    Point(int x = 0, int y = 0) : x(x), y(y) {}

    friend bool operator<(const Point& lhs, const Point& rhs) {
        if (lhs.x != rhs.x) {
            return lhs.x < rhs.x;
        }
        return lhs.y < rhs.y;
    }
};
```

Важно е тази наредба да бъде **смислена и последователна**.
Ако няма естествен ред между обектите, по-добре да не дефинираме `<`.

---

## Операторът `<=>` (three-way comparison)

От C++20 има нов оператор `<=>`.

Той се нарича **three-way comparison operator** или **spaceship operator**.

Идеята му е да обобщи сравнението между два обекта.

---

### Какво прави `<=>`

Вместо да дефинираме поотделно:

- `<`
- `<=`
- `>`
- `>=`

можем да дефинираме само `<=>`, а компилаторът да генерира останалите сравнения.

Пример:
```cpp
#include <compare>

class Point {
private:
    int x;
    int y;

public:
    auto operator<=>(const Point& other) const = default;
};
```

Тук компилаторът ще сравнява член по член:

- първо `x`
- после `y`

---

#### Какъв тип връща `<=>`
Tук е по-удобно да използваме `auto`, за да не пишем сложния тип ръчно:
```cpp
    auto operator<=>(const Point& other) const = default;
```

Но реално резултатът може да е от тип като:

- `std::strong_ordering`
- `std::weak_ordering`
- `std::partial_ordering`

- ако типът има пълна и ясна наредба, често се стига до `strong_ordering`
- ако има частична наредба, може да се получи друг тип

---
#### Как се използва резултата на `<=>`
В израз като 
```cpp
auto cmp = a <=> b
```
променливата `cmp` пази резултата от трипосочното сравнение между `a` и `b`.
Този резултат има три възможни смисъла:

`cmp < 0` — `a` е по-малко от `b`
`cmp == 0` — `a` е равно на `b`
`cmp > 0` — `a` е по-голямо от `b`

Когато пишем:
```cpp
if (auto cmp = lhs.x <=> rhs.x; cmp != 0) {
    return cmp;
}
```

това означава: сравни x координатите; ако те са различни, вече имаме окончателен резултат и го връщаме. Само ако са равни, продължаваме със сравнение по следващото поле.


#### Пример за ръчна реализация на `<=>`

```cpp
#include <compare>

class Point {
private:
    int x;
    int y;

public:
    Point(int x, int y) : x(x), y(y) {}

    friend std::strong_ordering operator<=>(const Point& lhs, const Point& rhs);
    friend bool operator==(const Point& lhs, const Point& rhs);
};

std::strong_ordering operator<=>(const Point& lhs, const Point& rhs) {
    // using the built-in <=> for int
    if (auto cmp = lhs.x <=> rhs.x; cmp != 0) {
        return cmp;
    }
    return lhs.y <=> rhs.y;
}

bool operator==(const Point& lhs, const Point& rhs) {
    return lhs.x == rhs.x && lhs.y == rhs.y;
}
```

Тук:

1. сравняваме `x`
2. ако са различни, връщаме резултата
3. иначе сравняваме `y`

---

#### Връзката между `<=>` и `==`

Важно уточнение: `<=>` и `==` са свързани, но не са едно и също.

Често е добра идея да дефинираме и:
```cpp
bool operator==(const Point& other) const = default;
```
Пример:
```cpp
#include <compare>

class Point {
private:
    int x;
    int y;

public:
    bool operator==(const Point& other) const = default;
    auto operator<=>(const Point& other) const = default;
};
```
Това е много кратък и удобен начин в C++20.

---

### Кога `<=>` е полезен

Полезен е когато:

- класът има естествен ред
- искаме да избегнем писане на много релационни оператори

Не е добър избор, ако:

- няма естествена наредба между обектите
- искаме специална логика само за някои сравнения

---

## Добри практики при предефиниране на оператори

### 1. Не предефинирай оператор само защото можеш

Операторът трябва да има **естествен и очакван смисъл**.

Лоша идея:
```cpp
a + b
```
да означава „изтрий файла“, „отвори мрежова връзка“ или друга напълно неочаквана операция.

### 2. Следвай интуицията на built-in типовете

Потребителят очаква:

- `+` да не променя операндите
- `+=` да променя левия обект
- `==` да връща `bool`
- `<<` да извежда към поток

### 3. Избягвай излишно повтаряне на код

Реализирай:

- `+` чрез `+=`
- `!=` чрез `==`
- други оператори чрез вече дефинирани базови оператори

### 4. Използвай `const`, когато трябва

Ако функцията не променя обекта, тя трябва да бъде `const`.

Пример:
```cpp
bool operator==(const Complex& other) const;
```

---

## Чести грешки

### Грешка 1: `operator+` променя левия аргумент
```cpp
Complex& operator+(const Complex& rhs); // лош дизайн
```
`+` трябва да връща нов резултат, а не да променя операндите.

### Грешка 2: `operator==` не е логически последователен

Ако `==` не проверява истинска еквивалентност, кодът става подвеждащ.

### Грешка 3: `operator<<` е член-функция

Това води до неестествен синтаксис:
```cpp
obj << std::cout;
```
вместо:
```cpp
std::cout << obj;
```
### Грешка 4: липса на `const`
```cpp
bool operator==(const Complex& other); // липсва const
```
Така няма да можем да сравняваме `const` обекти.

## Интересни примери за оператори:

### Conversion operators

Conversion операторите позволяват обект от даден клас да бъде преобразуван до друг тип. Те се дефинират в класа и описват какво означава обектът да се “разглежда като” стойност от друг тип.

Най-често се използват, когато даден клас е обвивка около някаква стойност и има естествено преобразуване към друг тип. Например клас може да се преобразува до bool, ако искаме да проверяваме дали обектът е в валидно състояние.

Тези оператори трябва да се използват внимателно, защото прекалено свободните преобразувания могат да направят кода неясен и да доведат до неочаквано поведение. Затова в много случаи е добра практика преобразуването да бъде explicit, за да се извършва само когато е поискано изрично.

```cpp
class Example {
public:
    explicit operator bool() const; // this allows to use objects of Example in if statements
    explicit operator int() const;
};
```

### Function call оператор `operator()`
Операторът `()` позволява обект да се извиква като функция. Класовете, които предефинират този оператор, често се наричат function objects, functors или callable objects.

Този оператор е полезен, когато искаме обект не само да пази данни, но и да изпълнява някакво действие при извикване. Това е удобно, когато поведението зависи от вътрешното състояние на обекта, или когато искаме да подадем обект там, където се очаква нещо “извикваемо”.

Типични случаи на употреба са:

- обекти, които работят като предикати, компаратори и подобни (филтриращи функции, сортиращи и тн.)
- обекти, които капсулират операция
- обекти със състояние, които се държат като функции

```cpp
class Example {
public:
    // function call operator
    void operator()();
    int operator()(int x) const;
};
```

### Member access оператор `operator->`

Операторът `->` позволява обект да се използва по подобен начин на указател. Той се предефинира, когато искаме обектът да предоставя достъп до членове на друг обект чрез синтаксис, подобен на този при указателите.

Най-често се използва в класове, които играят ролята на *smart pointers*, *итератори* или обвивки около друг обект. Идеята е потребителят на класа да може да работи удобно с вътрешния обект, без да мисли за детайлите по достъпа до него.

Този оператор е подходящ, когато класът логически “сочи към” друг обект и искаме използването му да наподобява работа с обикновен указател.

```cpp
class Example {
    SomeType* ptr = nullptr;
public:
    Example(SomeType* ptr);

    SomeType* operator->();
    const SomeType& operator->() const; // const overload, so that it can be called from const objects too
};
```
